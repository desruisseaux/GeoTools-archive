{@linkplain org.geotools.parameter.ParameterValue Parameter} implementation.

An explanation for this package is provided in the {@linkplain org.opengis.parameter OpenGIS® javadoc}. The remaining discussion on this page is specific to the Geotools implementation.

For our implementation of OperationalParameter (as returned by ParameterValue.getDescriptor()) we make use the classname ParameterDescriptor (rather than following the confusing OperationalParameter naming provided by geoapi).

We are targeting this class as a replacement for DataStore.PARAM. To provide pairity we need to account for the following features:

  • parse( text ) - parsing a text string into the parameter value
  • text() - parsable representation of value

These additional capabilities of our ParameterValue are designed to ease the creation of parsing and user interface routines. These do not represent a persistence systems as such.
