This package provides implementations for building general {@linkplain org.opengis.referencing.operation.MathTransform MathTransform} from a list of {@linkplain org.geotools.referencing.operation.builder.MappedPosition MappedPosition}.

This package should be used when we have two datasets that should fit to each other but they don't (unknown coordinate reference system may be the reason). When we are able to specify the points that are the same on both datasets ({@linkplain org.geotools.referencing.operation.builder.MappedPosition MappedPosition}) we should transform our data by various ways. This package provides implementations for building several transformation methods. The builders are:

  • {@linkplain org.geotools.referencing.operation.builder.RubberSheetBuilder RubberSheetBuilder}
  • {@linkplain org.geotools.referencing.operation.builder.BursaWolfTransformBuilder BursaWolfTransformBuilder}
  • {@linkplain org.geotools.referencing.operation.builder.ProjectiveTransformBuilder ProjectiveTransfrmBuilder}
  • {@linkplain org.geotools.referencing.operation.builder.AffineTransformBuilder AffineTransformBuilder}
  • {@linkplain org.geotools.referencing.operation.builder.SimilarTransformBuilder SimilarTransformBuilder}

where the least square method is applied on the errors between target points and transformed source points (in that case that we have more identical points then the necessary number).
